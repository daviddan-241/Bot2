# Deploy LeadFlow AI to Render

LeadFlow AI is ready for Render as a single Node web service. The Express backend serves the Vite production build in `NODE_ENV=production`.

## Recommended Render setup with persistent SQLite

Use `render.yaml` in this project.

It defines:

- Node web service
- `npm run render-build` build command
- `npm start` start command
- `/health` health check
- persistent disk mounted at `/var/data`
- SQLite database path `/var/data/leadflow.sqlite`

> Note: persistent disks may require a paid Render instance. If you deploy on a fully free instance without a disk, SQLite data can be lost on restart/redeploy.

## Steps

1. Push this folder to a GitHub repo.
2. In Render, click **New +** → **Blueprint**.
3. Connect the repo.
4. Render will detect `render.yaml`.
5. Deploy.

## Required environment variables

`render.yaml` already creates these:

```txt
NODE_ENV=production
PORT=10000
JWT_SECRET=<generated by Render>
DB_PATH=/var/data/leadflow.sqlite
DEFAULT_COUNTRY=US
EMAIL_QUEUE_DELAY_MS=2500
```

After your Render URL is created, update this environment variable if needed:

```txt
CORS_ORIGIN=https://your-render-service.onrender.com
```

For same-origin production use, the frontend calls the backend with relative API URLs.

## Optional provider setup

You can add provider credentials inside the app from **Connections**, so you do not need to put these in Render env variables:

- Gmail OAuth or SMTP
- Google Places / Serper / Brave / Bing / OpenStreetMap
- Ollama / Groq / HuggingFace
- WhatsApp Cloud API

## Free/ephemeral alternative

`render-free-ephemeral.yaml` is included for experiments on Render Free.

Warning: it uses local SQLite storage and may lose data after redeploy/restart. Use only for testing.
